PCIe

要学明白PCIe，有人建议先学明白PCI

那我就先学明白PCI吧！

PCI

PCI总线的重要特点

  • 地址空间隔离：HOST主桥将处理器地址转换位PCI总线地址
  • PCI设备使用的地址可以由软件动态分配，基地址位于每个PCI设备或者PCI桥的配置空间
  • 挂载在同一条PCI总线上的PCI设备共享该总线，PCI设备仲裁得到总线使用权后才能开始数据传输
  • PCI设备可以通过INTx信号向处理器提出中断请求。也可以通过MSI机制

PCI总线的主要结构

  • HOST主桥
    • PCI设备可以通过HOST主桥来访问主存储器，即执行DMA操作
    • 处理器也通过HOST主桥访问PCI设备
  • PCI总线
  • PCI设备
    • 主设备，从设备，主从设备，桥设备
    • 对PCI设备配置空间的访问是从上游总线到下游总线，数据传输可以双向进行
  • HOST处理器
    • HOST处理器通过操作HOST主桥的寄存器来管理PCI设备

PCI总线的信号定义

  • 同步总线
  • 地址/数据信号 控制信号 仲裁信号 中断信号
  • 地址/数据信号
    • AD[31:0]地址和数据复用信号，事务启动后第一个周期传输PCI总线域的地址，然后传输一到多个数据拍
    • PAR信号是AD和C/BE信号的奇偶校验信号
    • C/BE[3:0]，地址周期定义事务类型，数据周期对应字节选通

  • 接口控制信号
    • FRAME：主设备发出。PCI设备仲裁得到总线使用权时，拉低；结束总线事务，拉高
    • IRDY：主设备发出。写事务：数据已经在AD上有效；读事务：数据可以被发送到AD上。低电平有效
    • TRDY：从设备发出。写事务：目标设备已经准备好在AD上接收；读事务：数据已经发送到AD上。低电平有效
    • STOP：从设备发出
    • IDSEL：HOST主桥发出或者PCI桥转发，使用该信号选取PCI目标设备。
      存储器读写操作根据地址译码（决定访问哪个设备）
      而配置空间读写采用总线号，设备号和寄存器号译码
    • DEVSEL：从设备发出，通知PCI主设备，PCI总线上有它请求的从设备，但并不代表从设备可以和主设备进行数据交换了（这个工作由TRDY完成）
    • LOCK：主设备发出，将目标设备的某个存储器/IO资源锁定，防止其他PCI主设备访问，直到原主设备将资源释放。用的少，影响总线速率
  • 仲裁信号

    PCI主设备先置REQ有效，总线仲裁器会将指定的PCI主设备的GNT信号置位,然后PCI主设备就可以拉低FRAME启动它的数据传输了。
  • 中断请求信号
    INT（A, B, C, D)四个电平中断信号，PCI设备向处理器提交中断请求，拉低有效。处理器清除PCI设备的中断请求后，PCI设备将其拉高，结束中断请求。
  • 一些错误指示信号

PCI总线的存储器读写事务

  • 采用地址译码进行数据传输，采用ID译码进行配置信息读写
  • 时序图：

PCI主设备发送REQ，获得GNT

主设备在时钟沿拉低FRAME，传输第一拍地址和控制信息。如果是配置信息读写，也要拉低DEVSEL

主设备把数据准备好（拉低IRDY），从设备看情况使得TRDY有效，二者均有效那么完成一次握手。

数据传输结束后，将FRAME拉高，其他控制信号也复位为初始状态。

  • posted: PCI桥即可完成事务并释放总线，不用等到该事务的最终完成。适用于存储器写
  • non-posted: 必须等待事务彻底完成才能释放占用的一系列资源，适用于存储器读和IO读写，配置读写等
  • 处理器读写PCI设备
    • 一是一个PCI设备可以对应多个BAR，这些指的是PCI地址空间中的基地址，要和存储器地址进行转换。与BAR寄存器对应的PCI总线地址空间被称为BAR空间。在bar空间里可以存放IO地址空间和存储器地址空间
    • 原文中提到的存储器读写操作分别是posted和non-posted的，注意操作流程即可
  • PCI设备读写主存储器
    • PCI设备在进行DMA操作时，需要获取数据传输的目的地址（PCI域地址）和传输大小。储存这两个量的寄存器可以位于该PCI设备的BAR空间内。
    • PCI设备在进行DMA操作时，使用的是PCI地址。（PCI设备发起请求时，请求地址是PCI总线树上没有的地址，就有可能是存储器地址映射过来的PCI地址。但总之该地址还是位于PCI总线域中的）HOST主桥负责将它转换为存储器地址。存储器地址到PCI地址的映射是由HOST主桥完成的，可以只进行一部分存储器地址的映射，以保护存储器空间。
    • DMA读写操作，也比较容易理解
  • Delayed传输方式（针对non-posted事务的高占用）
    • 思想：PCI桥收到来自HOST主桥的non-posted事务时，如果认为该事务在16个周期内做不完，那么就进入delayed传输方式，否则保持普通传输方式。
    • 而delayed传输方式精髓在于，请求接收方接收到请求时，把请求往下传的同时，给请求发送方一个retry信号使得它过段时间再发起相同的请求，并释放上游总线。
    • 请求的完成信息在上游后续发起的第n次retry请求后返回。之后该级上游就不再会retry了。
    • 这种方法能够释放一部分总线时间，但是需要添加结构进行retry响应和重传请求。而且效率有改进但不多。

PCI总线的中断机制

很好的一节，使得我理解软硬件交互

  • 中断信号与处理器中断控制器的连接方式：因为主要用INTA信号，最好保证PCI设备与中断控制器引脚一一对应
  • 中断路由表：不同设备号的PCI设备，其INTX引脚和PCI总线的INTX引脚怎么连接？然后PCI总线的INTX再和处理器中断控制器的IRQXXX信号相连
  • 中断请求的同步
    • 试想一个PCI设备发起的DMA写操作，PCI设备把写数据发完就认为事务完成辣！然后发出一个INTA信号。但此时数据还没有被写入主存储器，直接用这个中断信号进入中断服务程序会有问题。
    • 交给软件来办：进中断后软件读PCI设备中断状态寄存器，判断原因后再操作DMA写的数据。这个操作包含一次读PCI设备的事务，利用PCI总线的序来保证DMA写一定已经完成。
    • 但如果这个操作没进行也不太可能出问题，因为从PCI设备提交中断到进入中断服务程序时间较长，足够DMA写的实际完成了。
  • MSI中断机制
    • 核心是向HOST处理器指定的一个存储器地址写包含有中断向量号的数据。HOST主桥会将该请求翻译为中断请求，提交给处理器。
    • 这样处理器可以得知是哪个PCI设备发起了中断，也不存在异步INTX的问题了，更加顺利的直接进入对应的中断服务程序。

PCI-X总线简介

  • split总线事务（对于non-posted)
    • 请求者向完成者发起读事务
    • 每经过一个桥，桥在转发该请求时，都会先返回一个split response结束这次事务，释放总线
    • 到达完成者后，完成者也会返回split response,并且记下请求者ID。
    • 完成者准备好数据，重新申请总线，组成完成报文。然后发送的完成报文根据报文附带的请求者ID路由，最终真正到达请求者，完成事务。
    • 特征：请求是地址路由的，完成报文是ID路由的
  • 传输协议和PCI的主要区别
    • 先锁存指令，下一个周期进行指令译码，增加总线频率
    • 可以不进行Cache一致性操作（这是什么？）
    • 支持乱序
  • 基于数据块的突发传输
    • 一次突发读写是一个以上的ADB（allowable disconnect boundary)

PCI总线的桥与配置

存储器域与PCI总线域

  • CPU域，也可以当作存储器域，CPU可以直接访问到的地址

  • 外部设备域

  • 处理器域是存储器域+外部设备域

  • 地址转换

    • PCI总线地址空间（存储在各自的bar中）在初始化时映射为存储器域的存储器地址空间，之后才能正确寻址
    • 反向映射（PCI总线域到存储器域），必须进行，且PCI设备只能访问这些被映射的空间
    • 不同PCI总线域可以拥有相同的地址，但含义不同，被映射的存储器地址也不同

HOST主桥（powerPC)

  • 确定PCI设备的ID

    • 总线号：HOST主桥遍历PCI总线树
    • 设备号：IDSEL信号与地址线的的连接方式决定
    • 功能号：每个设备有若干个功能
    • 寄存器号：PCI设备的具体哪个配置空间

  • 访问配置寄存器

    • 访问过程：处理器对HOST主桥中的CFG_ADDR和CFG_DATA寄存器进行访问，HOST主桥会将该访问转换为PCI配置读写请求。注意两个域之间大小端的转换

  • 地址转换(存储器域到PCI域)

    • 流程：32位软件地址->41位虚拟地址->36位物理地址->判断是不是在PCI地址空间->根据POWBAR确定在哪个窗口->根据POTAR转换为64位的PCI地址

  • 地址转换（PCI域到存储器域）

    • 转换方法类似
    • 注意peer to peer访问，也是通过INBOUND寄存器转换地址的

  • 能够被处理器和PCI设备同时访问的地址，一定在PCI域和存储器域中都有映射

  • HOST主桥（x86)简要提到

PCI桥与PCI设备的配置空间

  • PCI桥是透明桥：这是因为PCI桥的配置空间在系统遍历PCI总线树的时候进行配置
  • 非透明桥：使用PCI桥将一个处理器系统挂载在另一个处理器系统

PCI agent设备的配置空间

  • 通常由硬件在复位后读取EEPROM里的数据，对PCI设备的配置空间进行初始化
  • interrupt line 是中断向量号
  • interrpt pin是PCI设备使用的中断引脚编号
  • BAR：硬件先去写每个bar对应空间的大小/IO或存储器空间/是否支持预取；软件获得这个初始化信息；然后再将基地址写入该寄存器
  • 其他寄存器，用到的时候查阅即可

PCI桥的配置空间

  • PCI桥里的BAR表示该桥的私有寄存器，大多数桥里面是没有的
  • 总线号三个：最高级总线号，下一级总线号，上一级总线号；软件遍历PCI总线树时候设置
  • 两个status寄存器：一个status记录PCI桥作为PCI设备的status；第二个记录下一级总线的status
  • 两个latency timer（超时机制）:一个记录向下一级总线发出事务的延时，另一个记录向上一级总线发出事务的延时
  • IObase和limit：PCI子树IO基地址和大小
  • 存储器base和limit：一样的含义，但是大小至少为1MB
  • 预读base和limit
  • 桥控制寄存器，用到的时候查阅即可

PCI总线的配置

  • TYPE 00 访问直接与HOST主桥/PCI桥相连的PCI设备/PCI桥
  • TYPE 01 访问至少穿越一个PCI桥，访问没有与其直接相连的PCI设备或者PCI桥
  • TYPE01 的请求必定会被转换为TYPE00 的配置请求
  • CFG_ADDR与PCI总线AD信号的对应关系非常简单
  • 请求转换过程：01->01->…->01->00
  • 设备号分配：有一个固定的分配规则，每个PCI设备的IDSEL和AD信号有一个对应关系

PCI总线的数据交换

存储器地址和PCI地址的转换

    • 按照DFS算法顺序，是从PCI设备31和PCI设备32开始，倒着往上分配空间

PCI设备的数据传递

  • 正向译码：HOST主桥或者PCI桥发起请求，PCI设备看看请求的地址是不是在自己的bar空间内
  • 负向译码：HOST主桥或者PCI桥发起请求，三个周期后没有得到任何响应，则该器件被动接收该事务
  • PCI桥的一些特征（可选支持）
    • Combining
      • 将多个地址连续且大小为多个DW的存储器写事务合并为一个突发的事务
    • Merging
      • 将针对同一个DW不同字节的存储器写事务合并
    • Collapsing
